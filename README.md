# My Portfolio Project

このプロジェクトは、フロントエンドとバックエンドを分離した一人開発用ポートフォリオです。
「確実に動かす力」と「デバッグのしやすさ」を重視した構成にしています。

## 開発ポリシー：一人開発用「dist管理」運用

通常、ビルド成果物（`dist`）は Git 管理から外しますが、本プロジェクトでは以下の理由から **`dist` を Git に含めて管理** しています。

- **デプロイの確実性:** ローカルで動作確認したバイナリをそのままサーバーへ送ることで、本番環境でのビルドエラー（リソース不足や環境差異）を回避する。
- **一人開発の利点:** 競合（コンフリクト）が発生しないため、シンプルさを優先。
- **ロールバックの速さ:** 万が一の際、Git の履歴を戻すだけで即座に以前の正常な表示に戻せる。

## 主要コマンド（frontend/）

`frontend` フォルダ内で実行する主要なスクリプトです。

| コマンド                 | 用途             | 動作の詳細                                                                           |
| :----------------------- | :--------------- | :----------------------------------------------------------------------------------- |
| `npm run dev`            | **作業時**       | CSSビルド・HTMLコピー・ローカルサーバーを同時監視（並列実行）。                      |
| `npm run build`          | **ビルド確認**   | 監視なしで一度だけ全ファイルをビルドし、`dist` を最新にする。                        |
| `npm run deploy-prepare` | **保存・公開前** | **【最重要】** ビルド実行後、自動で `git add .` と `commit` を行い、保存漏れを防ぐ。 |

## セキュリティと環境制御（学んだこと）

- **.gitignore の役割分担:** ルート、frontend、backend でそれぞれ必要なものを守る（`.env` は絶対に公開しない）。
- **デバッグ力:** サーバーログ（`ENOENT` や `Missing script`）から、実行ディレクトリの間違いやスクリプトの未定義を特定し解決した。
- **ネットワーク設定:** ポートやTLS、環境変数を調整し、サーバー同士を繋げる土台を構築。
